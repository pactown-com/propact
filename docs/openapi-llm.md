# OpenAPI-LLM Integration

Propact integrates with [openapi-llm](https://github.com/Superflows-AI/openapi-llm) to enable semantic matching for any web service, even those without official OpenAPI specifications.

## 🌐 What is openapi-llm?

openapi-llm is a browser extension that:
- Monitors network traffic in your browser
- Generates OpenAPI specs from real API calls
- Enhances descriptions with LLM-generated natural language
- Creates dynamic specs for any web application

## 🚀 Workflow Integration

```
1. Browser → openapi-llm → spec.json (OpenAPI with LLM descriptions)
2. propact file.md --openapi spec.json --base-url "https://app.com"
3. Propact: embeddings + LLM → auto-match endpoint
4. Send → response → MD
```

## 📦 Installation

1. Install openapi-llm browser extension
2. Install Propact with semantic support:
```bash
pip install propact[semantic]
```

## 🔧 Usage

### Using Generated Spec
```bash
# Use spec generated by openapi-llm
propact README.md --openapi notion-openapi.json --base-url "https://api.notion.com/v1"
```

### Browser Session Integration
```bash
# Reference browser session (future feature)
propact README.md --openapi-llm-session ABC123 --base-url "https://app.com"
```

### Dynamic Spec Generation
```bash
# Generate spec dynamically from URL
propact README.md --base-url "https://unknown-app.com" --generate-spec --llm-key $OPENAI_KEY
```

## 📝 Example: Notion API

### 1. Generate Spec with openapi-llm
- Open notion.so in browser with openapi-llm extension
- Navigate through pages, databases, and features
- Export generated OpenAPI spec

### 2. Enhanced Descriptions
The generated spec includes LLM-enhanced descriptions:
```json
{
  "paths": {
    "/v1/pages": {
      "post": {
        "summary": "Create page",
        "description": "Create a new page in Notion",
        "x-llm-intent": "Create a new page with rich text content, blocks, and optional cover image or icon",
        "x-enhanced-description": "Create page Create a new page in Notion Create a new page with rich text content, blocks, and optional cover image or icon createPage"
      }
    }
  }
}
```

### 3. Use with Propact
```markdown
# Create Notion Database Entry

![task-screenshot.png](task-screenshot.png)

Create a new task entry in the project database with the following details:
- Title: "Fix Propact semantic matching accuracy"
- Status: In Progress
- Priority: High
```

```bash
propact README.md --openapi notion-openapi.json --base-url "https://api.notion.com/v1"
```

Output:
```
🧠 Top 3 semantic matches:
  1. POST /v1/pages (score: 0.419)
  2. POST /v1/databases (score: 0.406)
  3. POST /v1/comments (score: 0.093)

✓ Selected: POST /v1/pages
```

## 🔍 Importer Class

Propact provides `OpenAPILLMImporter` for handling generated specs:

```python
from propact.importer import OpenAPILLMImporter

# Import and enhance spec
spec = OpenAPILLMImporter.import_browser_spec("notion-openapi.json")

# Check if LLM enhanced
is_enhanced = OpenAPILLMImporter.is_llm_enhanced(spec)

# Extract LLM descriptions
descriptions = OpenAPILLMImporter.extract_llm_descriptions(spec)
```

## 🛠️ Advanced Features

### Dynamic Spec Generation
```python
from propact.dynamic_spec import generate_spec_from_url

# Generate spec from any URL
spec = await generate_spec_from_url(
    url="https://app.example.com",
    llm_key="your-openai-key"
)
```

### Session Management
```python
from propact.importer import OpenAPILLMImporter

# Get spec URL from session
session_id = "abc123"
spec_url = OpenAPILLMImporter.from_browser_session(session_id)
# Returns: "https://openapi-llm.superflows.ai/sessions/abc123/spec.json"
```
